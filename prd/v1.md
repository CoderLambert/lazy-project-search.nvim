# LazyVim Project Search PRD 与当前实现整理

## 一、项目定位

**LazyVim Project Search** 是一个基于 **LazyVim + Snacks Picker** 的项目搜索预设系统。

它要解决的问题是：

不同项目有不同的代码组织规则。比如你的 React 项目里经常需要搜索：

```text
hooks/use-claw-history-panel.ts
queryKey
className
TODO / FIXME
env 使用
NestJS Controller / Service
Vue defineProps / defineEmits
```

如果每次都手动 grep，会很低效。

所以我们希望把这些高频搜索沉淀为 **项目级搜索规则**，并通过一个统一入口打开。

---

# 二、核心目标

## 1. 用户体验目标

最终用户只需要记住一个快捷键：

```text
<leader>sP
```

也就是：

```text
空格 s P
```

第一次使用时：

```text
检测当前项目
自动生成规则文件
打开规则文件供用户编辑
```

后续使用时：

```text
打开项目搜索面板
左侧显示搜索规则
右侧预览规则说明
回车执行对应搜索
```

# 三、需求清单

## P0 必须支持

| 功能      | 说明                                  |
| ------- | ----------------------------------- |
| 统一入口    | 使用 `<leader>sP` 打开项目搜索面板            |
| 自动初始化   | 当前项目没有规则时，自动生成规则文件                  |
| 不污染源码   | 规则文件不放项目根目录                         |
| 项目级规则   | 每个项目拥有独立搜索规则                        |
| JSON 配置 | 搜索规则使用 JSON 配置                      |
| 可编辑     | 支持直接编辑当前项目规则                        |
| 可重置     | 支持重新根据模板生成规则                        |
| 可预览     | 搜索面板右侧显示完整规则说明                      |
| 多规则类型   | 支持 `files` / `grep` / `files_regex` |
| 模板生成    | 支持 common / react / vue / nest 基础模板 |

# 四、规则文件设计

## 1. 存储位置

规则文件不放在项目根目录，而是放到 Neovim 数据目录：

```text
~/.local/share/nvim/project-search/rules/<project-hash>.json
```

这样不会污染源码，也不会误提交到 Git。

---

## 2. 规则 JSON 示例

```json
{
  "version": 1,
  "meta": {
    "projectRoot": "/home/lambert/githubRepos/your-project",
    "template": "react",
    "createdAt": "2026-05-20T00:00:00Z",
    "note": "This file is stored outside your project."
  },
  "presets": [
    {
      "id": "react.hooks.kebab",
      "name": "Hooks: use-* kebab files",
      "description": "查找 hooks/use-xxx.ts 这类 kebab-case hook 文件，例如 hooks/use-claw-history-panel.ts。",
      "type": "files_regex",
      "regex": "(^|/)hooks/use-[a-z0-9-]+\\.(ts|tsx|js|jsx)$",
      "dirs": ["src", "app", "packages"],
      "exclude": ["node_modules", "dist", "build", ".next"]
    },
    {
      "id": "react.queryKey",
      "name": "React: queryKey",
      "description": "搜索 TanStack Query 的 queryKey。",
      "type": "grep",
      "search": "queryKey",
      "dirs": ["src", "app"],
      "glob": ["*.ts", "*.tsx"]
    }
  ]
}
```

# 五、规则类型设计

## 1. files

按目录查找文件。

```json
{
  "name": "Files: src",
  "type": "files",
  "cwd": "src"
}
```

适合：

```text
快速进入 src
快速进入 components
快速进入 hooks
快速进入 routes
```

---

## 2. grep

按文件内容搜索。

```json
{
  "name": "React: queryKey",
  "type": "grep",
  "search": "queryKey",
  "dirs": ["src", "app"],
  "glob": ["*.ts", "*.tsx"]
}
```

适合：

```text
搜索 queryKey
搜索 className
搜索 TODO / FIXME
搜索 env 使用
搜索 Controller / Injectable
```

---

## 3. files_regex

按文件路径正则搜索。

```json
{
  "name": "Hooks: use-* kebab files",
  "type": "files_regex",
  "regex": "(^|/)hooks/use-[a-z0-9-]+\\.(ts|tsx|js|jsx)$",
  "dirs": ["src", "app", "packages"]
}
```

适合你的项目规则：

```text
hooks/use-claw-history-panel.ts
hooks/use-user-info.ts
hooks/use-permission.ts
```

- src
- app
- packages

  "id": "react.hooks.kebab",

# 九、安装前置条件

## 1. 启用 Snacks Picker

进入 LazyVim：

```vim
:LazyExtras
```

启用：

```text
editor.snacks_picker
```

然后执行：

```vim
:Lazy sync
```

---

## 2. 安装系统依赖

Ubuntu / Linux Mint：

```bash
sudo apt update
sudo apt install -y ripgrep fd-find

mkdir -p ~/.local/bin
ln -sf "$(command -v fdfind)" ~/.local/bin/fd
```

检查：

```bash
fd --version
rg --version
```

# 十二、下一步优化方向

## 1. 更好的规则管理

支持：

```text
启用规则
禁用规则
删除规则
复制规则
从当前搜索保存为规则
```

---

## 2. 支持规则变量

例如：

```json
{
  "vars": {
    "sourceDirs": ["src", "app", "packages"],
    "tsFiles": ["*.ts", "*.tsx"]
  },
  "presets": [
    {
      "name": "React: queryKey",
      "type": "grep",
      "search": "queryKey",
      "dirs": "$sourceDirs",
      "glob": "$tsFiles"
    }
  ]
}
```

---

## 3. 支持规则继承

例如：

```json
{
  "extends": ["common", "react"],
  "presets": [
    {
      "name": "Project: claw hooks",
      "type": "files_regex",
      "regex": "(^|/)hooks/use-claw-[a-z0-9-]+\\.(ts|tsx)$",
      "dirs": ["src"]
    }
  ]
}
```

---

## 4. 支持项目索引

维护：

```text
~/.local/share/nvim/project-search/index.json
```

记录所有初始化过的项目：

```json
{
  "projects": [
    {
      "name": "openclaw-ui",
      "root": "/home/lambert/githubRepos/openclaw-ui",
      "template": "react",
      "lastUsedAt": "2026-05-20T10:00:00Z"
    }
  ]
}
```

后续可以提供：

```vim
:ProjectSearchProjects
```

查看所有项目规则。

# 十三、当前结论

这套功能已经从简单快捷键，升级成了一个完整的 **LazyVim 项目搜索规则系统**。

当前最推荐路线是：

```text
第一步：先用单文件版本验证体验
第二步：拆成本地插件
第三步：在多个项目中验证规则模板
第四步：稳定后再考虑开源
```

核心价值是：

```text
用 JSON 管理项目搜索习惯
用 Snacks Picker 执行搜索
用统一面板聚合规则
用项目维度保存配置
不污染源码
不破坏 LazyVim 默认体验
```

我建议下一步直接把它拆成 `lazy-project-search.nvim` 本地插件，这样后面继续扩展会更清晰。
