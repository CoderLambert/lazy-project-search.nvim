# v0.2 目标：把它整理成一个本地 LazyVim 插件

插件名建议：

```text
lazy-project-search.nvim
```

定位：

```text
基于 Snacks Picker 的项目级搜索规则系统
```

目标体验不变：

```text
<leader>sP
```

打开项目搜索面板。

但内部从单文件升级为模块化结构，后续好维护、好扩展。

---

# 一、v0.2 要解决的问题

当前单文件版本虽然能用，但有几个问题：

| 问题   | 说明                                   |
| ---- | ------------------------------------ |
| 文件过大 | 所有逻辑都在一个 `project-search.lua` 里      |
| 不好扩展 | detector、storage、runner、picker 都混在一起 |
| 不像插件 | 没有标准 `setup()`、配置项、模块边界              |
| 不好复用 | 换机器或分享给别人不方便                         |
| 不好维护 | 后续加变量、规则继承、项目索引会变乱                   |

所以 v0.2 要做的是：

```text
从 LazyVim 单文件配置
升级成本地 Neovim 插件
```

# 三、模块职责

| 文件              | 职责                                  |
| --------------- | ----------------------------------- |
| `init.lua`      | 插件入口，提供 `setup()`                   |
| `config.lua`    | 默认配置                                |
| `util.lua`      | 通用工具函数                              |
| `detector.lua`  | 检测 React / Vue / Nest 项目            |
| `storage.lua`   | 管理规则文件路径、读写 JSON                    |
| `templates.lua` | 内置 common / react / vue / nest 模板   |
| `runner.lua`    | 执行 `files` / `grep` / `files_regex` |
| `picker.lua`    | 打开 Snacks Picker 面板和预览              |
| `commands.lua`  | 注册命令                                |
| `health.lua`    | 健康检查                                |

这样后续加功能不会乱。

# 四、LazyVim 中如何加载本地插件

新建：

```bash
nvim ~/.config/nvim/lua/plugins/lazy-project-search.lua
```

写入：

```lua
return {
  {
    dir = "~/githubRepos/lazy-project-search.nvim",
    name = "lazy-project-search.nvim",
    dependencies = {
      "folke/snacks.nvim",
    },
    opts = {
      keymap = "<leader>sP",
      storage_dir = vim.fn.stdpath("data") .. "/project-search/rules",
      auto_init = true,
    },
  },
}
```

然后执行：

```vim
:Lazy sync
```

# 五、插件配置项设计

`setup()` 支持这些配置：

```lua
{
  keymap = "<leader>sP",

  auto_init = true,

  storage_dir = vim.fn.stdpath("data") .. "/project-search/rules",

  default_excludes = {
    ".git",
    "node_modules",
    "dist",
    "build",
    ".next",
    ".nuxt",
    "coverage",
  },

  templates = {
    common = true,
    react = true,
    vue = true,
    nest = true,
  },

  picker = {
    title = "Project Search",
    layout = "default",
  },
}
```

这样以后用户可以覆盖默认行为。

初始化当前项目规则。


强制重新生成当前项目规则。


```vim
:ProjectSearchPath
```

# 九、v0.3 可以继续做的优化

v0.2 先完成插件化。v0.3 再做高级功能。

## 1. 规则变量

例如：

```json
{
  "vars": {
    "sourceDirs": ["src", "app", "packages"],
    "tsFiles": ["*.ts", "*.tsx"]
  },
  "presets": [
    {
      "name": "React: queryKey",
      "type": "grep",
      "search": "queryKey",
      "dirs": "$sourceDirs",
      "glob": "$tsFiles"
    }
  ]
}
```

---

## 2. 规则继承

例如：

```json
{
  "extends": ["common", "react"],
  "presets": [
    {
      "name": "Project: claw hooks",
      "type": "files_regex",
      "regex": "(^|/)hooks/use-claw-[a-z0-9-]+\\.(ts|tsx)$",
      "dirs": ["src"]
    }
  ]
}
```

---

## 3. 项目索引

维护：

```text
~/.local/share/nvim/project-search/index.json
```

记录所有初始化过规则的项目。

后续可以实现：

```vim
:ProjectSearchProjects
```

用于查看、编辑、删除历史项目规则。

---

## 4. 面板内规则管理

在搜索面板中支持：

```text
Enter      执行规则
Ctrl-e     编辑规则文件
Ctrl-r     重置规则
Ctrl-y     复制规则 JSON
Ctrl-d     禁用当前规则
```

---

## 5. 从当前搜索保存为规则

例如你手动搜索了：

```text
use-claw-history-panel
```

然后执行：

```vim
:ProjectSearchSaveGrep
```

输入规则名后自动保存到当前项目规则 JSON。

这个功能会非常实用。
